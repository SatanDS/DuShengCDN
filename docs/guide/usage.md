# 基础使用

你会学到：DuShengCDN 中网站配置、源站、证书、版本、节点和观测分别是什么，以及日常使用时应按什么顺序操作。

DuShengCDN 不直接在线修改节点上的代理服务配置。你在管理端修改的是控制面数据；只有发布并激活新版本后，节点程序才会拉取完整配置并应用到节点。

## 核心概念

| 概念 | 说明 |
| --- | --- |
| 网站配置 | 反向代理配置的聚合对象，一条网站配置可以绑定一个或多个域名 |
| 主域名 | `domains` 列表中的第一个域名，用作该网站的主要展示域名 |
| 源站 | 被反向代理访问的源站地址，例如 `http://10.0.0.10:8080` |
| 配置版本 | 一次发布生成的完整代理服务配置快照，历史版本不可变 |
| 激活版本 | 当前全局生效的配置版本，所有节点默认消费同一份激活版本 |
| 节点程序 | 节点侧进程，也叫 Agent，负责注册、心跳、同步、校验、重载和失败回滚 |
| 节点池 | 节点调度分组，用于自动解析目标选择和缓存运行时操作范围 |
| 公网 IP 池 | 节点可用于 A/AAAA DNS 记录的公网地址列表 |

## 推荐操作顺序

日常发布一条反向代理配置时，推荐按这个顺序：

1. 在左侧「节点和IP池」确认至少有一个节点在线，并维护节点池、公网 IP 池、池内权重和排空状态。
2. 新增或选择源站地址。
3. 新增网站配置，填写域名、源站和站点级配置。
4. 如需负载均衡，先在左侧「Cloudflare 账号」准备 Cloudflare 账号，再进入对应网站配置的「负载均衡」分区选择解析模式、默认节点池或多节点智能解析策略。
5. 如需 HTTPS，在左侧「域名资产」进入「TLS 证书」模块上传或申请证书，并按域名绑定。
6. 预览配置或查看变更摘要。
7. 在左侧「发布版本」发布并激活新版本。
8. 在节点详情和左侧「应用记录」中确认应用结果。

## 创建网站配置

网站配置至少需要：

| 字段 | 要求 |
| --- | --- |
| 网站名称 | 业务唯一标识；未显式填写时通常可使用主域名 |
| 域名 | 至少一个域名，第一项为主域名；任一域名全局只能属于一个网站 |
| 源站地址 | 合法的 `http://` 或 `https://` 地址 |
| 启用状态 | 只有启用的网站配置会参与发布渲染 |

示例：

| 字段 | 示例 |
| --- | --- |
| 网站名称 | `docs` |
| 域名 | `docs.example.com` |
| 源站地址 | `http://10.0.0.10:8080` |
| 回源 Host | `docs.internal.example.com` |

源站地址规则：

* 单源站可以携带 base path 或 query，例如 `https://app.example.com/base?from=dushengcdn`。
* 多源站用于负载均衡时，每个源站必须是纯 `scheme://host[:port]`。
* 多源站在同一规则内应使用一致协议。

## 节点池与自动解析

节点池是控制面调度元数据，不改变全局单激活版本。常见做法：

* 在左侧「节点和IP池」里把同一组边缘节点放到同一个节点池，例如 `default`、`asia` 或 `edge-hk`。
* 为节点维护公网 IP 池；自动解析只会选择符合记录类型的公网 IPv4/IPv6。
* 节点里的「池内权重」只影响同一个节点池内多个节点的选择比例；HK 池、EU 池这种跨节点池比例要在网站配置「负载均衡」里的节点池权重中设置。
* 开启“参与自动调度”后，该节点可被自动解析选择；开启“排空模式”后，自动解析与缓存运行时操作都会跳过该节点。
* 新建网站和编辑“反向代理”时，默认节点池会自动列出已创建节点的节点池名称，也可以手动输入新的节点池。
* 网站配置列表里，未启用多节点智能解析的网站会显示默认池和池内真实节点；启用多节点智能解析后会显示“负载均衡”和该网站配置的调度节点池。编辑“反向代理”时，节点池旁边会同步显示当前池内真实节点，便于确认选择的池是否有可用节点。编辑“负载均衡”的多节点智能解析时，每个节点池会同步列出池内真实节点并默认全选；取消勾选后，该池只使用保留勾选的节点参与解析。
* 网站配置中的默认节点池是默认承载池：未启用多节点智能解析时，自动解析从这个池选择公网 IP；缓存清理/预热也下发到这个池。
* 多节点智能解析只决定 A/AAAA 返回哪些边缘 IP。启用后，解析目标按「负载均衡」里的多个节点池策略计算，网站默认节点池仍保留为缓存、攻击防护回退和运行时兜底，两者不冲突。

Cloudflare 自动解析支持：

* A/AAAA 记录自动选择一个或多个在线节点公网 IP。
* `记录内容` 手动填写时，可用逗号、空格或换行提供多个 A/AAAA 内容。
* 自动模式可按“健康优先（冷却防抖）”、节点池内权重或负载感知评分选择，并支持同步多个 A/AAAA 目标。健康优先只判断节点是否在线、代理服务是否健康、是否允许调度、是否排空以及最近心跳时间；它不使用处理器、内存或连接数评分。冷却防抖表示旧目标仍健康且切换冷却时间未到时暂时保持旧解析答案，避免解析频繁来回抖动。
* 需要跨多个节点池分流时，在网站详情的「负载均衡」里启用多节点智能解析，点击 `+` 逐行添加真实节点池、池权重、可选国家代码和来源网段，例如池名 `hk`、池权重 `80`、国家代码 `HK,TW`、来源网段 `203.0.113.0/24`。来源网段会优先于国家代码匹配。
* `按压力优先` 模式会结合节点池内权重、池权重、代理服务当前连接数、处理器和内存快照评分；在同一区域可配置最大连接数、最大处理器压力和最大内存压力，超过任一阈值的节点会被跳过。默认只使用最近 `120` 秒内的节点指标，可在设置页的「本地解析运行参数」调整节点状态数据有效时间；没有新鲜指标的节点不会被直接剔除，但只会在有指标候选不足时作为兜底目标。
* 在本地自建解析模式下，选择按权重优先或按压力优先时，会按来源 IP/ECS 生成稳定分流桶，所以类似 HK 池权重 80、EU 池权重 20 的配置会在不同来源桶之间形成接近 8:2 的解析答案分布；Cloudflare 模式只同步一组静态记录，不具备逐查询来源分流。
* 多节点智能解析会保存最近一次实际目标和期望目标，旧目标仍健康且处于冷却期时不会频繁切换解析答案；本地自建解析模式下，DNS 响应端会先在内存中维护逐来源防抖状态，再随心跳批量回传面板。
* 当前 Cloudflare 模式是后台重算并同步 A/AAAA 记录，受解析缓存时间和运营商 DNS 缓存影响；如需按每次用户来源实时返回不同 IP，需要切换到本地自建解析并部署 DNS 响应端。
* Cloudflare API Token 可直接填写原始 Token，也兼容 `Bearer ...` 和包含 `api_token` / `apiToken` / `token` 的 JSON。
* `常态开启 Cloudflare 代理` 只表示正常状态下是否同步橙云；攻击自动防护会在攻击期临时覆盖解析目标，恢复后仍回到这个常态设置。
* `攻击防护模式` 选择 `自动` 后，需要再选择防护提供方：`Cloudflare` 会在攻击期暂停多节点智能解析、临时回到网站默认节点池并强制同步 Cloudflare 橙云；`自定义清洗池` 会在攻击期暂停多节点智能解析、只返回指定节点/IP 池里的在线公网 IP，并关闭 Cloudflare 橙云代理。最近 5 分钟请求量和错误率恢复到阈值以下后，下一轮巡检会回到正常的固定内容、默认节点池或多节点智能解析。

本地自建解析的实时多节点智能解析模式：

* 左侧「本地自建解析」用于创建托管域名、维护 NS/SOA 和静态记录，并创建 DNS 响应端接入令牌。
* 新增 A/AAAA 静态记录时，记录值会以一个输入框一个 IP 的方式填写，可点 `+` 增加多个 IP；MX 记录的优先级数字越小越优先，常见主邮件服务器填 `10`，备用服务器填 `20`。
* 左侧「本地自建解析」的「迁移向导」会列出 Cloudflare 模式网站候选，检查是否已有匹配托管域名、在线响应端、公网 UDP/TCP 53 探测、公网可达响应端调度快照未过期且版本一致、同名静态记录冲突、当前节点池或多节点策略是否能返回可用边缘 IP，以及策略里显式配置的来源国家、来源网段是否都有可用边缘 IP；满足条件时可直接点击「一键切换」，也可以跳转到网站详情手动调整。一键切换成功后，向导会自动刷新网站解析模式、执行托管域名委派检查、探测在线响应端，并按当前快照执行 global 与来源国家模拟复测。
* 网站详情的「负载均衡」里仍可把 `解析模式` 从 `Cloudflare 同步` 切换为 `本地自建解析`，再绑定已启用的托管域名；启用状态下保存时会执行和迁移向导一致的 DNS 响应端在线、公网探测、快照新鲜度/一致性、可用边缘 IP 和显式来源作用域预检，未启用草稿可先保存再补齐节点与响应端。
* 网站域名必须属于所选托管域名；`*.example.com` 这类通配符域名在本地自建解析下只匹配单级子域名，例如 `api.example.com` 会命中，`deep.api.example.com` 不会命中；切换后 Cloudflare 账号、橙云和 Cloudflare 防护提供方不再参与该网站解析回答。本地自建解析查询面无法直接开启 Cloudflare 小黄云，若需要攻击期切到自有清洗 IP，应使用自定义清洗池。
* 域名需要在注册商处把 NS 委派到 DuShengCDN DNS 响应端。
* 托管域名详情里的「委派检查」可以对比注册商当前公网 NS 与面板配置的 NS，并列出缺失或额外的 NS。
* 如果 NS 名称位于当前托管域名内，例如 `ns1.example.com` 服务 `example.com`，需要在注册商配置 Glue/主机记录；面板会在检查结果中提示。
* 迁移向导的「切换后复测」只会检查和提示，不会修改注册商 NS；如果复测显示委派部分匹配、不匹配或需要 Glue，仍需到注册商完成调整后再次检查。
* DNS 响应端会在每次 A/AAAA 查询时根据来源 IP/ECS、来源网段、国家代码、来源分流桶、节点池权重、节点健康和负载评分返回边缘 IP。按压力优先查询会优先在带有新鲜指标的节点中选择；当匹配节点池里暂无新鲜指标候选时，才回退到无指标节点，避免节点漏报指标时被误判为空闲。设置页「本地解析运行参数」里的「按响应端探测结果筛选节点」默认关闭；开启后，本地自建解析快照和 DNS 响应端实时调度只会使用仍有新鲜成功响应端探测结果的边缘节点，并把探测健康比例、过期比例和平均 RTT 转成有边界的探测质量系数，参与权重或负载感知评分。切换到本地自建解析前，控制面会先复用同一组选点规则确认 global 和策略里声明的来源国家、来源网段至少有一个可返回目标；如果节点池名写错、节点无公网 IP、A/AAAA 类型不匹配、节点离线、代理服务不健康、排空、负载阈值过严，或开启探测筛选后节点缺少新鲜成功探测，迁移向导、一键切换和网站详情启用保存都会阻断。
* 已由网站配置接管的动态解析域名，不要再在托管域名静态记录里创建同名同类型 `A`/`AAAA` 或同名 `CNAME`；控制面会直接拒绝保存。需要保留 `TXT`、`MX`、`NS`、`SOA` 等其它类型记录时可以继续在托管域名里维护。
* DNS 响应端会从面板拉取只读调度快照，本地缓存最后一次有效快照，并从快照恢复最近可用的防抖状态；面板短暂不可用时仍可继续回答静态记录。
* 左侧「本地自建解析」会展示最近 24 小时的查询量、查询趋势、SERVFAIL/NXDOMAIN 趋势、响应端快照一致性、响应端查询延迟、可用率、错误率、最近公网探测健康状态、节点多地探测通过率/RTT、GeoIP 国家库加载状态、来源作用域、响应端/托管域名/站点维度、返回目标分布和当前调度状态，适合确认多节点智能解析是否按预期把 `cidr:203.0.113.0/24`、`country:HK`、`country:DE`、`global|bucket:42` 等来源分配到 HK、EU 等节点池，并发现多响应端快照版本不一致或快照过期问题。
* 「GSLB 调度状态」会按网站、记录类型和来源作用域展示当前实际目标、期望目标、最近评估时间和防抖状态；当显示「冷却中」时，表示期望目标已经变化，但旧目标仍健康或仍处于防抖冷却期，所以解析答案暂时保持当前目标。
* 左侧「本地自建解析」的「GSLB 调度模拟」可选择本地自建解析站点、记录类型、来源国家代码和来源 IP，预演当前快照会返回哪些边缘 IP、缓存时间和来源作用域，并查看节点池匹配、候选节点、跳过节点、负载指标时间、原因，以及每个边缘节点到 DNS 响应端的节点多地探测通过数和 RTT；当本次模拟没有可返回目标时，也会保留匹配节点池、候选节点、跳过节点和无目标原因，便于确认是节点池、健康状态、公网 IP、负载阈值还是探测门槛导致；该操作只读，不会写入真实防抖状态。
* 在 DNS 响应端列表可点击「探测」，由面板对该响应端公网地址发起 UDP/TCP 53 的 SOA 查询，确认解析可达性、RTT、RCODE 和错误信息；最近一次探测结果会保存在响应端列表和可用性面板中。
* 在线节点会从心跳设置里收到少量在线 DNS 响应端的探测目标，并从各自节点发起 UDP/TCP `53` 查询；本地自建解析的响应端可用性面板会按节点展示探测点、节点池、UDP/TCP 可达性、平均 RTT、最大 RTT 和失败原因。
* 迁移向导、一键切换和网站详情启用保存都会要求公网可达的在线 DNS 响应端均已拉取未过期调度快照，且这些可服务响应端的快照版本一致，避免响应端只会连回面板、但公网递归 DNS 无法访问 `53` 端口，或部分 NS 尚未拿到可回答动态记录的同一份快照时误切换或误保存。
* 响应端延迟来自 DNS 响应端本地处理真实查询的聚合耗时；节点多地探测 RTT 则表示各边缘节点到响应端 NS 的主动探测耗时。超过新鲜度窗口的节点多地探测会标记为“探测过期”，仍保留明细但不计入健康通过率；「GSLB 调度模拟」里的节点探测摘要用于解释边缘节点到各 NS 的可达性。默认情况下多点探测只用于观测与排障；只有开启「按响应端探测结果筛选节点」后，才会影响本地自建解析的多节点选点，且探测质量系数会被限制在 `0.25` 到 `1.0` 之间，避免少量失败样本或 RTT 抖动完全压倒节点池权重、节点权重和负载感知评分。
* 未配置本地 GeoIP 库时，国家代码为空，调度作用域会回退到 `global`；响应端列表和可用性面板会显示 GeoIP 是否加载、数据库路径和最近加载错误，便于排查国家代码节点池不命中的原因。
* 这种模式不依赖 Cloudflare API，也不支持 Cloudflare 橙云代理；需要自行保证 DNS 响应端的公网可达和高可用。
* 生产环境至少部署两个 DNS 响应端，并放行 UDP/TCP 53。
* 详细设计见 [自建权威 DNS 与 GSLB 调度规划](../design/authoritative-dns-gslb.md)。

## 缓存清理与预热

网站缓存策略仍通过发布版本生效；缓存清理与预热属于运行时操作：

* “清理全部缓存”会通过 WebSocket 下发到网站节点池内的在线 Agent，由 Agent 清理受限的 OpenResty 缓存目录。
* “预热站点首页”会让目标节点主动请求当前网站域名首页。
* 运行时操作不生成配置版本，也不替代发布流程。
* 节点详情会展示缓存命中率、HIT/MISS/STALE 计数、回源 5xx 次数与平均回源耗时。

## 管理源站

源站是轻量目录，用来复用常见源站地址。网站配置关联源站后，仍会保存可渲染的 `origin_url` 快照，确保历史配置版本可以独立回放。

推荐做法：

* 把经常复用的内部服务地址维护为源站。
* 修改源站目录后，检查已发布的网站配置是否需要同步更新源站快照。
* 发布前使用预览或 diff 确认渲染结果。

## 启用 HTTPS

HTTPS 按域名绑定证书，而不是按整个网站统一强制启用。

操作顺序：

1. 在左侧「域名资产」进入「TLS 证书」模块上传或托管证书。
2. 申请证书时可选择 Cloudflare 账号验证，也可选择本地自建解析的托管域名验证；本地方式会临时写入 `_acme-challenge` TXT 记录并在验证后清理，证书里的域名必须属于所选托管域名。
3. 进入网站配置，为需要 HTTPS 的域名选择证书。
4. 未绑定证书的域名会保留 HTTP，不会被自动放入 `443 ssl` server 块。
5. 发布并激活新版本。

如果一个网站包含多个域名，Server 发布时会按证书分组渲染 HTTPS 配置，同时保持这些域名属于同一份网站快照。

## 发布、激活与回滚

标准链路：

```text
修改配置 -> 预览 / diff -> 发布 -> 生成完整版本 -> 激活版本 -> Agent 拉取 -> 本地应用 -> 上报结果
```

发布时 Server 会读取全部启用的网站配置、OpenResty 主配置模板、性能参数、缓存参数和证书资源，生成完整配置并计算 `checksum`。

回滚不是修改历史版本，而是重新激活旧版本。Agent 发现激活版本变化后，会按普通同步流程拉取并应用。

## 查看节点与观测

节点页面适合回答三个问题：

| 问题 | 查看位置 |
| --- | --- |
| 节点是否在线 | 节点列表或节点详情 |
| 当前运行哪个版本 | 节点详情中的当前版本 |
| 最近一次应用是否成功 | 应用记录 |

左侧「观测计量」合并访问日志与计量统计，适合查看最近 24 小时的每站点流量、每节点流量、缓存命中率、回源流量、状态码分布、TOP URL / TOP IP / TOP 地区、带宽峰值 P95 和节点可用率。

访问分析和资源快照用于基础观测。DuShengCDN 只保留受控时间窗口内的访问明细，不定位为通用日志平台。如果需要长期日志检索，应接入独立日志系统。回源流量依赖新版访问日志中的 `upstream_response_length` 字段，恶意请求防护命中原因依赖新版访问日志中的 `reason` 字段；升级 Agent 并发布新配置后才会从新日志开始计量和显示，历史日志不会补算这些字段。
恶意请求防护使用观察模式时，请求会继续放行，命中的规则可到「观测计量」->「明细日志」里查看状态码旁的 `!`；拦截模式下同样在这里查看拦截规则和原因。

## 左侧主目录建议

左侧主目录只放独立资源或独立运维视角，避免把必须依附具体网站配置的功能拆散：

| 主目录 | 适合处理 |
| --- | --- |
| 节点和IP池 | 边缘节点、节点池、公网 IP 池、池内权重、调度开关、排空状态 |
| 网站配置 | 域名、源站、负载均衡、缓存、恶意请求防护、计算验证防护、地区限制和认证等站点级配置 |
| 域名资产 | 可复用域名资产、默认证书绑定，以及 TLS 证书上传、ACME 申请、续期和证书详情 |
| 发布版本 | 预览、发布、激活和回滚 |
| 应用记录 | 节点应用配置的成功、警告和失败记录 |
| 观测计量 | 访问日志、IP 维度、站点/节点流量、缓存命中、回源流量和带宽 P95 |
| 代理服务配置 | 代理服务连接、反代缓冲、压缩、缓存基础设施和主配置模板 |

负载均衡、缓存策略、计算验证防护、恶意请求防护、地区限制和认证配置当前仍属于具体网站配置；它们影响代理服务配置渲染和发布版本，不单独作为主目录，避免脱离站点后产生配置归属冲突。管理端在「网站配置」详情页内提供这些站点级配置分区，可通过站点详情页的分区导航进入。

## 常见场景

### 新增一个内部服务反代

1. 确认源站服务可从边缘节点访问。
2. 在管理端新增网站配置。
3. 填写域名，例如 `app.example.com`。
4. 填写源站，例如 `http://10.0.0.20:8080`。
5. 发布并激活版本。
6. 在 Agent 节点或浏览器访问域名验证。

### 给已有域名启用 HTTPS

1. 准备覆盖该域名的证书。
2. 在「域名资产」的「TLS 证书」模块上传或创建证书记录。
3. 回到网站配置，为对应域名选择证书。
4. 发布并激活版本。
5. 用浏览器或 `curl -I https://your-domain` 验证证书链和状态码。

### 回滚一次失败发布

1. 打开配置版本页面。
2. 找到上一个已知可用版本。
3. 重新激活该版本。
4. 查看节点应用记录，确认节点已应用旧版本。
5. 修正配置后再发布新版本。

## 推荐实践

* 生产环境显式配置 `SESSION_SECRET`，并优先使用 PostgreSQL。
* 修改端口、DSN、Token、密码等本地部署参数时，优先写入 `dushengcdn_server/.env` 或 Compose override，避免后续 `git pull` 被本地 `docker-compose.yaml` 修改阻塞。
* 升级前备份数据库和上传目录，升级后确认 Server 日志、节点状态、应用记录和当前激活版本。
* 修改网站配置后先看预览或 diff，再发布。
* 每次发布后检查节点详情与应用记录。
* 多节点部署时保持节点到面板的网络路径稳定。
* 不在节点上手动修改 DuShengCDN 托管的代理服务配置文件；下次发布会覆盖这些文件。
